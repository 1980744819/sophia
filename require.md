# Sophia 在线考试系统需求文档

## 1. 项目概述

Sophia 是一个在线考试系统，主要用于市总工会组织的工会社工技能竞赛。该系统将支持实时在线答题、防作弊功能以及考试结束后立即出成绩的需求。

### 1.1 背景

市总工会计划在3月份组织全市390名工会社工进行技能竞赛，竞赛结果将影响社工们的绩效奖金和推选到省里参加更高层级比赛的资格。因此，领导对此次考试高度重视，需要一个可靠、公平的在线考试平台。

### 1.2 目标

- 提供便捷的在线考试环境，支持社工通过扫码或链接进入考试
- 确保考试公平性，实现防作弊功能
- 实现考试时间控制，确保在规定时间内完成答题
- 提供即时成绩反馈，考试结束后立即出成绩
- 支持考试数据的后台管理和查看

## 2. 功能需求

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 考生 | 无需注册，通过信息验证进入 | 参加考试、查看个人成绩 |
| 管理员 | 后台账号 | 管理题库、查看所有考生成绩、导出考试数据 |

### 2.2 考生端功能

#### 2.2.1 考试入口
- 支持通过扫描二维码或点击链接进入考试
- 初始界面需填写个人信息：姓名、工号、手机号、身份证号、工作单位
- 信息验证通过后进入考试界面

#### 2.2.2 考试过程
- 题库规模：50-100题
- 题型：客观题（选择题、判断题等）
- 考试时间：45分钟或60分钟（可配置）
- 时间倒计时显示
- 自动保存答题进度
- 时间结束后自动提交试卷并关闭考试

#### 2.2.3 成绩查询
- 考试完成后立即显示成绩
- 支持查看个人答题情况和正确答案

### 2.3 管理端功能

#### 2.3.1 题库管理
- 支持题目添加、编辑、删除
- 支持题目分类管理
- 支持批量导入题目

#### 2.3.2 考试管理
- 支持设置考试时间、题目数量等参数
- 支持考试开始和结束时间设置
- 支持考试状态监控

#### 2.3.3 成绩管理
- 实时查看所有考生成绩
- 支持成绩导出（Excel格式）
- 支持按部门、分数等条件筛选

### 2.4 防作弊功能

- 防截屏功能
- 防同账号异地登录
- 考试过程中页面离开检测

## 3. 非功能需求

### 3.1 性能需求

- 系统需支持至少500人同时在线考试
- 页面加载时间不超过3秒
- 成绩计算响应时间不超过2秒

### 3.2 安全需求

- 数据传输加密
- 敏感信息（如身份证号）加密存储
- 防止SQL注入等常见安全漏洞

### 3.3 兼容性需求

- 支持主流浏览器：Chrome、Firefox、Safari、Edge
- 支持移动端和PC端访问

### 3.4 可靠性需求

- 系统可用性达到99.9%
- 数据定期备份
- 具备异常情况处理机制

## 4. 系统流程

### 4.1 考生流程

1. 考生通过扫码或链接进入考试系统
2. 填写个人信息（姓名、工号、手机号、身份证号、工作单位）
3. 信息验证通过后，系统显示考试须知和倒计时
4. 考试开始，考生进行答题
5. 考试过程中，系统实时倒计时
6. 考生完成答题后点击提交，或系统在时间结束后自动提交
7. 系统立即计算并显示成绩
8. 考生查看成绩详情

### 4.2 管理员流程

1. 管理员登录后台系统
2. 配置考试参数（时间、题目数量等）
3. 管理题库（添加、编辑、删除题目）
4. 监控考试过程
5. 查看和导出考生成绩

## 5. 技术架构

### 5.1 前端技术

- 响应式Web设计
- 前端框架：Vue.js/React
- 状态管理：Vuex/Redux
- UI组件库：Element UI/Ant Design

### 5.2 后端技术

- 服务端框架：Node.js/Express
- 数据库：MongoDB/MySQL
- 认证：JWT

### 5.3 部署方案

- 云服务器部署
- CDN加速
- 负载均衡（可选）

#### 6.1.1 考生信息

| 字段名 | 数据类型 | 描述 |
|--------|----------|------|
| id | String | 唯一标识符 |
| name | String | 姓名 |
| employeeId | String | 工号 |
| phone | String | 手机号 |
| idCard | String | 身份证号 |
| workUnit | String | 工作单位 |
| createTime | Date | 创建时间 |

#### 6.1.2 题目信息

| 字段名 | 数据类型 | 描述 |
|--------|----------|------|
| id | String | 唯一标识符 |
| content | String | 题目内容 |

| options | Array | 选项（选择题） |
| correctAnswer | String/Array | 正确答案 |

| 字段名 | 数据类型 | 描述 |

| id | String | 唯一标识符 |
| examineeId | String | 考生ID |

### 6.2 数据存储

- 考生信息和考试记录存储在数据库中
- 题目信息存储在数据库中
- 备份策略：每日备份

## 8. 风险评估

### 8.1 潜在风险

1. **技术风险**：系统稳定性和并发处理能力
2. **安全风险**：防作弊措施可能被破解
3. **时间风险**：开发时间紧张，可能影响系统质量
4. **用户风险**：部分考生可能不熟悉在线考试操作

### 8.2 应对措施

1. **技术风险**：进行充分的性能测试，确保系统能够支持390人同时在线考试
2. **安全风险**：多重防作弊措施并用，定期更新防作弊技术
3. **时间风险**：合理安排开发计划，优先实现核心功能
4. **用户风险**：提供详细的操作指南，提前进行模拟测试

## 9. 验收标准

1. **功能验收**：所有需求文档中描述的功能均已实现
2. **性能验收**：系统能够支持390人同时在线考试，页面加载时间不超过3秒
3. **安全验收**：防作弊功能有效，数据传输和存储安全
4. **用户体验**：界面友好，操作简便，响应及时
